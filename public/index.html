API Documentation

Toys API

Http method Route Description Parameters/Request Body
Get /toys Returns a list of toys according to the query parameters
  • page:page number to return list of toys(defualt is 1).
    https://nodejs-project.cyclic.app/toys?page=1
  • PerPage:numbers of toys returns in every page (defualt is 5).
    https://nodejs-project.cyclic.app/toys?perPage=5
  • sort:returns property by sort (defualt is "id")
    https://nodejs-project.cyclic.app/toys?sort=name
  • reverse:if reverse set to yes, sorts in ascending order (defualt: descending ).
    https://nodejs-project.cyclic.app/toys?sort=price&reverse=yes
Get /toys?s= returns specific property by search
  • s: search query to match specific property
    https://nodejs-project.cyclic.app/toys?s=toy
Get toys/category?cat= returns toys by category
  • cat:the name of the category to retrieve toys for.
    https://nodejs-project.cyclic.app/toys/category?cat=Accessories
Get /toys/price Returns a list of toys filtered by price range
  • min:minimum price to toy(defualt: 0)
    https://nodejs-project.cyclic.app/toys/price?min=50
  • max:maximum price to toy (defualt: infinity)
    https://nodejs-project.cyclic.app/toys/price?max=30

    minimum and maximum price:
    https://nodejs-project.cyclic.app/toys/price?min=20&max=50

Get /toys/single/:id returns single toy by his id
  • id:get a toy by his id
    https://nodejs-project.cyclic.app/toys/single/645910564027041851a872ce
Post /toys add new toy (token is required!)
  • reqBody: Toy data to be added in the following schema
  • schema:
    {
    name: String,
    info: String,
    category: String,
    price: Number,
    img_url: String
    }
Put /toys/:id update toy(token is required!)
  • id: the id of the toy to update
  • reqBody: Toy data to be updated in the following schema.
  • schema:
    {
    name: String,
    info: String,
    category: String,
    price: Number,
    img_url: String
    }
Delete /toys/:id delete toy(token is required!)
  • id: the id of the toy to delete

Users API

Http method Route Description Parameters/Request Body
Get /users Test endpoint to check if the server is running null
Get /users/userInfo Get the information for the authenticated user (requires authentication) null
Post /users Create a new user {
name: String,
email: String,
password: String
}
Post /users/login Authenticate a user and receive a JWT token in response {
email: String,
password: String
}

Users/Admin Token Abillities

Admin User
Has the ability to add new toys to the system Has the ability to add new toys to the system same as admin
Can edit all information related to any toy in the system Can only edit information related to toys they have created or are associated with their account
Has the ability to delete all toys from the system Can only delete toys that they have created or are associated with their account
